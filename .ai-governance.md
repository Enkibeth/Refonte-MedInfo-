# .ai-governance — Règles pour agents IA (Codex / Claude Code)

Lire `docs/README.md` puis `docs/01_REGULATION.md` AVANT toute action.

## Invariants non négociables
1. Aucune feature de : triage symptomatique perso, calculateur clinique interprétatif,
   interaction médicamenteuse avec avis, diagnostic différentiel individualisé.
2. Aucun stockage de donnée santé identifiable hors HDS.
3. Aucun marketing/copy contredisant l'intended purpose (docs/01_REGULATION §1).
4. Une seule app Expo. Un seul package.json. Pas de second langage backend. Pas de fichier hors structure (docs/02_ARCHITECTURE §2).
5. Prompts = artefacts versionnés sous contrat (docs/04_CHATBOT §2). Jamais éditables en prod.
6. Refus = defense-in-depth 3 couches (classifieur + prompt + validation sortie).

## Workflow PR
- Branche : ai/<agent>/<feature>/<short-desc>
- Mettre à jour docs/CHANGELOG_AI.md (fichiers, but, impact réglementaire none/potential/confirmed, rollback)
- Les 5 gates CI doivent passer (compliance-grep, refusal-regression, rls-isolation, prompt-contract, rag-license)
- Toute décision d'archi/produit → ADR dans docs/DECISIONS/

## En cas de doute réglementaire
Ne pas implémenter. Ouvrir un ADR "Proposed" et demander arbitrage humain (Hugo).
